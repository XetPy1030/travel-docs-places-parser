# Подсказки для агента / контрибьютора

## Назначение

Репозиторий — CLI-парсер DOCX → JSON с вызовами OpenRouter и поиском изображений. Основная логика в `main.py`.

## Окружение и секреты

- Ключ API: переменная **`OPENROUTER_API_KEY`**. Не коммитить `.env`; ориентир — `.env.example`.
- Зависимости: **Poetry** (`pyproject.toml`, `poetry.lock`).

## Ввод и вывод

- Вход: папка с подпапками районов и `.docx` внутри (`--input`).
- Справочник районов: опционально `--districts` (например `tatarstan_locations.json`).
- Вывод по умолчанию: каталог **`output/`** (`--output-dir`), итоговый файл по умолчанию `attractions.json` внутри него (`--output` / `-o`). Абсолютный `-o` не подмешивается к `--output-dir`.
- Возобновление: `--resume` использует `processing_state.json` в том же `--output-dir`.

## Что не раздувать без запроса

- Не добавлять тесты и объёмную документацию без явной просьбы пользователя.
- Менять только то, что нужно для задачи; стиль и структура как в существующем `main.py`.

## Соглашения репозитория

- Переносы строк и базовые отступы: `.editorconfig` и `.gitattributes`.
- Сгенерированные JSON и логи: см. `.gitignore` (в т.ч. `output/*`, кроме `output/.gitkeep`).
